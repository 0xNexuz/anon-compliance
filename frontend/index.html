How the demo works.

01

Generate proof data

The frontend creates commitments, a nullifier, and a proof payload for the selected compliance policy.

02

Verify ZK on Stellar

The backend calls the deployed Groth16 verifier contract and sends that verification to testnet so users can inspect the proof tx.

03

Record attestation

If verification passes, AnonCompliance records a nullifier and compliance attestation on Stellar testnet.

Current caveat: the Groth16 verifier is real, while the included circuit is the Stellar sample circuit. The product path is to replace it with an AnonCompliance-specific KYC, sanctions, or RWA eligibility circuit.
